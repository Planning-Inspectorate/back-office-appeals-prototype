Appeal a planning decision

Linked and related appeals

As a BO user I want to link / unlink appeals so that I can process cases in combination to align the decision process and economise CO/inspector time resource

Jira tickets: BOATD-25, BOATD-71

Version 3 {%- from "govuk/components/tag/macro.njk" import govukTag -%} {{govukTag({ text: "New", classes: "govuk-!-margin-left-2 govuk-tag--green" })}}

This version contains both linked and related appeals. I've kept them in the same space in the prototype as they go hand in hand and we should keep the pattern usage consistent. If you use the appeal ID 5176892 in either journey it will show the error for when the appeal is not found.

Assumptions:

  • That we will be able to query our database to see if a claim is in back office. If not we can check the horizon api to see if an appeal exists and return some basic information.

There are four possible variations of the page that is returned when trying to link to an appeal, they are:

  • Default - This is the screen if there are no existing links for the appeals.
  • Linking from a lead appeal - This is the screen if the appeal you are linking from is already a lead, therefore the appeal you're linking to can only be a child.
  • Linking to a lead appeal - This is the screen if the appeal you are linking to is already a lead, therefore the appeal you're linking from can only be a child.
  • Linking not possible - This is the screen if the appeal you are trying to link to is a child appeal, therefore cannot be linked to. (It can't have two leads.)

This version contains both linked and related appeals. I've kept them in the same space in the prototype as they go hand in hand and we should keep the pattern usage consistent.

Features:

  • ability to link claims
  • ability to set claims as a lead or child
  • ability to manage and unlink claims
  • ability to show claims as related
  • ability to manage related claims

Assumptions:

  • That we will be able to query our database to see if a claim is in back office, if not we can send them on the journey for horizon appeals.

What this doesn't do:

  • the prototype shows the links for adding multiple appeals but doesn't do it functionally
